SECTION 0: HOW TO READ THIS DOCUMENT
================================================================================
This document is the single source of truth for the PrepCreatine frontend.
It covers every page, every component, every interaction, every error state,
and every design decision. Read it fully before writing any code. Do not
hallucinate features not described here. Do not invent routes, components, or
behaviors that are not specified. When something is ambiguous, default to the
simpler, cleaner implementation.

This document uses the following notation:
  [REQUIRED]   — must be implemented exactly as described
  [OPTIONAL]   — implement if time allows, skip otherwise
  [FORBIDDEN]  — never do this under any circumstances
  [NOTE]       — important context or reasoning
  [DARK]       — dark mode specific instruction
  [A11Y]       — accessibility requirement
  [PERF]       — performance requirement
  [LOG]        — add a console.log or logger call here for debugging
  [NEW]        — added in v2.0 (did not exist in v1.0)
  [MOD]        — modified from v1.0
  [DEMO]       — demo-mode-specific behaviour

SECTION 1: TECH STACK (REQUIRED — DO NOT DEVIATE)
================================================================================

FRAMEWORK & BUILD
  - React 18 with Vite (NOT Create React App, NOT Next.js)
  - TypeScript strict mode enabled
  - React Router v6 for all routing

STATE MANAGEMENT
  - TanStack Query (React Query v5) for all server state and API calls
  - Zustand for global client state (UserContext, auth, theme, notifications)
  - React Hook Form for ALL forms — never use raw useState for form fields
  - No Redux, no MobX, no Context API for state that could go in Zustand

STYLING
  - Tailwind CSS v3 with custom theme extension (see Section 3)
  - tailwind-merge for conditional class merging
  - clsx for className logic
  - NO inline styles except for dynamic values that Tailwind cannot express
  - NO styled-components, NO CSS modules, NO emotion

ANIMATIONS
  - Framer Motion for all page transitions and component animations
  - CSS transitions for hover states (faster than JS)
  - NO GreenSock, NO anime.js

UI COMPONENTS
  - Lucide React for ALL icons — import individually (tree shaking)
  - react-day-picker for all date pickers
  - React Flow (@xyflow/react) for roadmap, concept maps, knowledge graph
  - react-virtual (TanStack Virtual) for virtualized lists
  - DOMPurify for sanitizing any HTML from user content or AI
  - react-hot-toast for all toast notifications
  - canvas-confetti for topic completion celebration [NEW: also used in SM-2]

NEW IN v2.0:                                                           [NEW]
  - @mediapipe/face_mesh for on-device gaze proctoring
    (lazy-loaded — only imported inside GazeProctor.tsx)
  - react-markdown + remark-gfm for chat message rendering (was already used)
  - recharts [OPTIONAL] for knowledge graph mastery bars if Chart.js feels heavy

HTTP & API
  - Axios with a custom instance (see Section 8) — never use raw fetch
  - All API calls go through TanStack Query hooks
  - Sentry (@sentry/react) for error monitoring

FORMS & VALIDATION
  - React Hook Form (always)
  - Zod for schema validation via @hookform/resolvers
  - All Zod schemas in /src/schemas/

PWA
  - Vite PWA plugin (vite-plugin-pwa) with Workbox
  - Service Worker registered automatically

FONTS
  - Poppins: weights 400, 500, 600, 700 via @fontsource/poppins
  - Inter: weights 400, 500 via @fontsource/inter
  - JetBrains Mono via @fontsource/jetbrains-mono
  - Load via @fontsource packages, not CDN

DEPLOYMENT TARGET
  - Vercel (primary)
  - All environment variables prefixed with VITE_
  - vercel.json must rewrite all routes → index.html

ENVIRONMENT VARIABLES                                                  [NEW]
  VITE_API_BASE_URL=http://localhost:8080
  VITE_DEMO_MODE=true          ← set to true for hackathon demo
  VITE_SENTRY_DSN=             ← optional, can be empty in demo

DEMO BANNER COMPONENT (components/demo/DemoBanner.tsx)                [NEW]

  Only rendered when useDemoMode() returns true.
  File: src/components/demo/DemoBanner.tsx

  import { useDemoMode } from '../../hooks/useDemoMode'
  import { client } from '../../api/client'
  import toast from 'react-hot-toast'

  export function DemoBanner() {
    const isDemo = useDemoMode()
    if (!isDemo) return null

    const handleReset = async () => {
      try {
        await client.get('/api/demo/reset')
        toast.success('Demo data reset. Refreshing...')
        setTimeout(() => window.location.reload(), 1500)
      } catch {
        toast.error('Reset failed. Try refreshing manually.')
      }
    }

    return (
      <div className="fixed bottom-0 left-0 right-0 z-50 h-11
                      bg-slate-900 border-t border-slate-700
                      flex items-center justify-between px-4 gap-3">
        <div className="flex items-center gap-2">
          <span className="w-2 h-2 bg-orange-400 rounded-full animate-pulse" />
          <span className="text-xs text-slate-300">
            Demo Mode · Arjun Sharma · JEE · 60 days to exam
          </span>
        </div>
        <button
          onClick={handleReset}
          className="text-xs text-slate-400 hover:text-white border
                     border-slate-600 rounded-lg px-3 py-1
                     transition-colors duration-150"
        >
          Reset demo
        </button>
      </div>
    )
  }

  export function useVoiceInput(onResult: (text: string) => void,
                                language: 'hi-IN' | 'en-IN' | 'en-US') {
    const [isRecording, setIsRecording] = useState(false)
    const recognitionRef = useRef<SpeechRecognition | null>(null)

    const isSupported = typeof window !== 'undefined' &&
                        ('SpeechRecognition' in window ||
                         'webkitSpeechRecognition' in window)

    const startRecording = () => {
      if (!isSupported) return
      const SpeechRecognition = window.SpeechRecognition ||
                                 window.webkitSpeechRecognition
      const recognition = new SpeechRecognition()
      recognition.lang = language      // 'hi-IN' for Hindi
      recognition.continuous = false
      recognition.interimResults = false

      recognition.onresult = (event) => {
        const transcript = event.results[0][0].transcript
        onResult(transcript)            // sets chat textarea value
        setIsRecording(false)
      }

      recognition.onerror = () => setIsRecording(false)
      recognition.onend   = () => setIsRecording(false)

      recognition.start()
      recognitionRef.current = recognition
      setIsRecording(true)
    }

    const stopRecording = () => {
      recognitionRef.current?.stop()
      setIsRecording(false)
    }

    return { isSupported, isRecording, startRecording, stopRecording }
  }

    ({ gaze_away: 0, tab_switch: 0, face_lost: 0 })

  Step 1: Get webcam stream

    Attach to <video ref={videoRef} autoPlay playsInline muted />

  Step 2: Load FaceMesh (lazy import)
    const facemesh = await import('@mediapipe/face_mesh')
    const faceMesh = new facemesh.FaceMesh({ ... })
    faceMesh.onResults(onResults)

  Step 3: Run inference every 500ms
    setInterval(() => {
      if (videoRef.current) faceMesh.send({ image: videoRef.current })
    }, 500)

  Step 4: onResults(results)
    if (!results.multiFaceLandmarks?.length) {
      // No face detected
      gazeEvents.current.face_lost++
      return
    }
    const landmarks = results.multiFaceLandmarks[0]
    // Check gaze direction using iris landmarks (index 468-472)
    // If iris X position < 0.3 or > 0.7: looking left/right (gaze away)
    const irisX = landmarks[468].x
    if (irisX < 0.3 || irisX > 0.7) gazeEvents.current.gaze_away++

  Step 5: Tab switch detection (browser native)
    document.addEventListener('visibilitychange', () => {
      if (document.hidden) gazeEvents.current.tab_switch++
    })

  export function getMasteryColorClass(mastery: number): string {
    if (mastery < 0.26) return 'bg-red-100 text-red-700'
    if (mastery < 0.51) return 'bg-amber-100 text-amber-700'
    if (mastery < 0.76) return 'bg-sky-100 text-sky-700'
    return 'bg-green-100 text-green-700'
  }

  export function getReviewDueLabel(nextReviewDate: string): string {
    const today = new Date()
    today.setHours(0, 0, 0, 0)
    const due = new Date(nextReviewDate)
    due.setHours(0, 0, 0, 0)
    const diff = Math.floor((due.getTime() - today.getTime()) / 86400000)
    if (diff < 0) return `Overdue by ${Math.abs(diff)} day${Math.abs(diff) > 1 ? 's' : ''}`
    if (diff === 0) return 'Review due today'
    if (diff <= 3) return `Review in ${diff} day${diff > 1 ? 's' : ''}`
    return `Review on ${due.toLocaleDateString('en-IN', { day: 'numeric', month: 'short' })}`
  }

  export function getReviewDueColorClass(nextReviewDate: string): string {
    const label = getReviewDueLabel(nextReviewDate)
    if (label.startsWith('Overdue')) return 'bg-red-100 text-red-700'
    if (label.includes('today'))  return 'bg-red-100 text-red-700'
    if (label.includes('in 1') || label.includes('in 2') || label.includes('in 3'))
      return 'bg-amber-100 text-amber-700'
    return 'bg-gray-100 text-gray-500'
  }
